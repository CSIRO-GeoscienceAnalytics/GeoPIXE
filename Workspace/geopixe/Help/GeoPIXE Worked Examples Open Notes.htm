Edit your ‘geopixe.conf’ file

Using the gedit command, open your GeoPIXE configuration file ‘geopixe.conf’ located in your home “~/.geopixe/” directory. The specific lines to edit are (assuming Demo data located at path /data/Demo/ - adapt these to your local Demo path):

 

1.       path   data   /data/Demo/                                                                                   default raw data directory

2.       path   analysis   /data/Demo/                                                                            default analysis results directory

3.       path   config   /data/Demo/Maia/Standards/config/                                      default standards config for Wizard Standards

4.       cluster   type   cores                                                                                            select multiple cores for parallel processing

5.       cluster   nodes   10                                                                                              use 10 cores in “cluster” mode

6.       default   device  MAIA_DEVICE                                                                    set Maia as the default GeoPIXE device

 

Path Bookmarks

After you have navigated to your Demo directory for data and/or analysis, Add this path to your Bookmarks in the GeoPIXE File-Requester, and Save these for later use. Read more about the GeoPIXE File-Requester in the GeoPIXE Users Guide (in your doc and /Help directories).

Path Translation

GeoPIXE can translate paths to a new location using the Path Translation tables, which are set-up in the File-Requester. You can define the base of some path and a new value for this base (for example, after you have copied it from the beamline computers, you might wish to translate the beamline path base to your local system path base).

 

 

Then, if a selected file path is not found, GeoPIXE will search for a translation of the old path in these Translation Tables. If the file is then found on one of these new translated paths, it is used from the new path.

A.  Spectrum Analysis and Fitting

The examples assume that the demo data has been loaded into a directory “Demo\”. You will need to navigate to this new path and save it as a Bookmark in the file requester for future reference.

Example A1:  Spectrum display

  • Loading spectra and display of spectra
  • Energy calibration and X-ray line identification
  • Escape peaks and pileup

 

Steps to try

  1. Load the PIXE spectrum file “Donut2x-2-whole.spec” from directory “Demo/CSIRO/Donut2x”.
    1. Find the “Demo” directory on your system in the File Requester (see tips provided by host). Click on “Add” under “Bookmarked Paths” to add it to your bookmarks list and hit “Save”.  Next time you try these examples you can double-click on this bookmark to access the Demo directory directly.
    2. Similarly, you can set-up a path translation “Translation Table” to automatically translate from old paths to the paths as you see them from your Training account.
  2. Set the View markers and expand the view using “Widen”.
    1. To select the View markers, select “View 0,1” in the marker droplist in the bottom-left corner.
    2. Set the right marker first, at the high-energy end (left mouse click), and then the left one (set View 2-42 keV).
    3. Perhaps drag the window larger to see more detail.
    4. Click on “Widen” to expand View range.
  3. View maximum detail in the vertical scale using “Auto” vertical scale.
    1. Hover mouse pointer over buttons and controls to see help on them.
  4. Experiment with re-sizing the Spectrum Display window, scrolling left and right through the spectrum using the horizontal scroll bar and changing the X scale (using “Expand”, “Shrink”, “Widen”, “Full”) and Y scale (“Up”, “Down”, “Log/Lin”).

 

X-ray Peak Identification

  1. Open the X-ray Identification window, if it’s not open, by clicking on the “?” button.
    1. If the X-ray Identification window is in “Mark Element” mode, select “Identify” mode.
    2. Click in the Identification window list to select the “Identify” marker on the spectrum, or select the “Identify” marker on the spectrum droplist.
    3. Click and drag on the spectrum to move the “Identify” marker.
    4. The nearest X-ray energy match is highlighted in the Identification list.
    5. Click on a line in the list to mark that energy in the spectrum (other lines for the selected element will be highlighted in the list).
  1. Change the relative intensity threshold using the slider in the Identification window (e.g. click to the right of the slider knob to increment the threshold by ~0.05). This hides weak lines.
  2. Select “Mark Element” in the Identification window and click on elements to mark their lines on the spectrum (e.g. Fe, Sr). Alternatively, click on the arrow buttons to increment/decrement atomic number (or use keyboard arrow keys).
  3. Step across atomic numbers Z from 25 to 60 to see elements present as K lines and Z from 73 to 83 for L lines. Beware peaks that are due to “Ge escapes” or “pileup”.

 

Escape and Pileup Peaks

  1. Data were collected using a Ge detector. Calculate and display Ge escape peaks using the menu “Display->Ge Escapes”.
    1. Escapes shown in violet are dominated in this case by the escapes of the Sr K lines. Note the presence of escapes due to both Ge Ka (9.88 keV) and Ge Kb (10.98 keV) energy deficits.
  1. Calculate pile-up using the menu “Display->Pileup->from single spectrum”.
    1. Note the presence of Fe Ka pile-up at 12.8 and 13.4 keV. This doesn’t seem to explain the spectrum peak near to 12.6 keV.
    2. However, there is also Pb present in the sample, which gives a 12.6 keV contribution due to the Lb1 line. Mark the Pb lines to see this.

 

Tips:

·         To set markers, select the type using the droplist (bottom left of Spectrum window), then click to the right of the displayed area to position the high marker, and then to the left to set the low marker. Click, hold and drag the markers to position them.

·         Many windows provide context-sensitive help tips in the text widget at the bottom of the window (e.g. Spectrum Display, GeoPIXE Image and X-ray Spectrum Fit).

o    Move the cursor over a widget to see help on it.

·         After resizing a window, click “Widen” again to view the View range expanded to the new window size.

·         To switch to the “Identify” marker again, just click in the element list in the X-ray Line Identification window, or use the Droplist on the left of the Spectrum Display window.

·         To make the lines marked for an element show the effects of absorption (which changes the relative intensities of an element’s lines) select the appropriate filter (200 µm Al) and detector (CSIRO #34).

·         This is most noticeable at low energy (e.g. mark Fe lines and change the “Filter” between 200 µm Al and 250 µm Be).

  3. Load the fit set-up file “null.pcm”, which has no elements selected.
    1. An estimate of background will be drawn in violet on the spectrum.

 

X-ray Peak Identification

  1. Click on elements in the periodic table to select elements seen in spectrum (e.g. Fe, Sr, Sn and Ba).
    1. Click once for K, twice for L shell, etc. – the colour of the button will change to reflect the of K lines (green), L lines (yellow), M lines (red), both K and L lines (light blue), etc.
    2. Note how selecting Sr K lines shows the Ge escape peaks as well.
  1. Use the X-ray Identification window to identify lines, or to mark lines for elements to help identify them.
    1. Step through atomic number using the scroll arrows.
  1. Select elements in the fit by either:
    1. Left clicking on the element in the periodic table in the X-ray Spectrum Fit window, or
    2. Right clicking on an element in the X-ray Identification window. (Note: this assumes K lines up to Ba, and L lines for La and heavier.)
  1. Work through identifying all green peaks.
    1. No, there is not Kr in this sample (collected in vacuum). So what is the 12.6 keV peak?

 

X-ray Spectrum Fitting

  1. Once you have identified most peaks, fit the spectrum using the “Fit: One” button in the X-ray Spectrum Fit window.
  2. Check out any green peaks that don’t seem to be explained by the elements you have selected.
    1. Did you see Ga and Ge?
    2. How’s the fit to the 12.6 keV peak?

                                                               i.      Fe pileup at 12.8 keV helps, but is not quite right.

                                                             ii.      What about Pb L lines? Click on Pb twice to select Pb L lines (now shown in yellow on the periodic table).

                                                           iii.      Click on the “Fit: One” button again.

    1. To see any elements you might have missed, load the set-up file “Donut2x.pcm”.

 

Fit results

  1. Open the Fit Results window using the Image Display menu “Window->Fit Results”.
    1. Results of the fit are shown in ppm (weight) or labelled as % (for weight %).
    2. You might want to drag the window larger to see more columns.
  2. View detection limits (ppm) by changing the drop-list from “Conc” to “MDL”.
    1. e.g. Fe is 4.5 wt% and Br is 3.5 ppm with a MDL of 0.3 ppm.

 

Tips:

·         Enable “Diff” on the Spectrum Display window to see the difference between the data and the background.

o    This can make weak lines more visible.

·         To disable an element in the fit (i.e. do not fit it, but still calculate MDL), “right-click” on it with the mouse in the periodic table in the X-ray Spectrum Fit window (element colour darkens). To re-enable it, “left-click”.

·         To clear an element entirely, “shift-left-click” on it, or keep left-clicking until it cycles back to grey (off).

Example A3:  Fitting a SXRF spectrum, Yield modelling

  • Spectrum fitting for quantitative analysis
  • SXRF yield modelling (simple detector)
  • This example assumes an average matrix composition. See the later section “Spectrum fitting using end-member analysis” for methods to iterate on sample composition for more quantitative fitting results.

 

Steps to try

  1. Load the spectrum file “Demo/APS/Pottery/sample24_2D.spec”.
  2. Set the view around the peaks and “Widen” with “Auto” vertical scale.
  3. Load the fit set-up file “null.pcm” from the Pottery directory into the X-ray Spectrum Fit window.
  4. Identify peaks and select them for fitting.
    1. Data used no additional filter, just a small “air” path, so select “none” or “Air 20 mm” filter in the X-ray Line Identification window to show correct relative intensities (makes a big difference for Ti and lighter elements).
    2. Note the peaks on the right are due to Compton (inelastic) scattering (18.4 keV) and elastic scattering (19 keV).
    3. This spectrum was collected with a Silicon-drift detector. Hence, it will only have (weak) Si-escape lines.
    4. Did you notice the hidden peak at 13.0 keV? What’s this due to?
    5. Load the file “pottery.pcm” to select all lines.

 

Yield Calculation

  1. Open the Yield Calculation window, by clicking on the “New” button adjacent to “Yields” on the X-ray Spectrum Fit window.
  2. Load the yield set-up file “pottery.lcm”. Note the selection of “Photons  (mono)” as “beam Particle”, an energy of 19 keV, the angles to the detector (90°) and the rotation of the target towards the detector (-45°).
    1. To get help on widgets, move the cursor over each and look at the help in the bottom pane.
    2. Note that the “Array” check-box is NOT set indicating that this is using a simple (small-solid-angle) detector model.
    3. To view this beam-target-detector geometry, click on the “?” button next to the Phi detector angle widget. Left-click and drag to rotate the displayed 3D view of the sample detector geometry.
  1. Experiment with other detector angles. Note that positive Theta is anti-clockwise as viewed from above. Use negative Theta for a detector on the other side. Use positive Alpha target tilt with a negative Theta (and negative Alpha with positive Theta), unless you want a transmission geometry.
    1. Close the geometry window and click on “?” with each change to view the geometry (it does not automatically update).
    2. Use the Rotation, Zoom, Translate modes on the geometry window to rotate and zoom the 3D view. Click and drag in the window to change the view for each mode.
  1. Note that Theta is the detector angle in the plane, while Phi is the out-of-plane angle, measured as an azimuthal rotation about the beam (Z) axis. Alpha is the target rotation, about the vertical axis, while Beta is a tilt about an axis across the face of the target.
    1. Use the context-sensitive help in the PIXE/SXRF Yield Calculation window for help on these angles, or experiment and use “?”
  1. Try Theta=80, Phi=20, Alpha=55, Beta=25, for a more extreme example.
    1. Close the geometry window and click “?” again to open it anew.

                                                               i.      The meaning of the various angles is now clearer.

                                                             ii.      Click and drag in the geometry window to rotate the 3D view.

    1. Is there anything “wrong” with this geometry?

                                                               i.      Now you have the detector looking at the BACK of the target.

                                                             ii.      If transmission mode is not desired, set Alpha to -55 and check the geometry again.

 

 

Beam-target-detector geometry for a = 55º (left; detector is ‘behind’ target) and a = -55 º (right; probably the way it should be).

 

  1. Re-load the yield set-up file “pottery.lcm” to restore geometry. Close any geometry windows.
  2. To calculate yields (as well as relative intensities), including secondary fluorescence modelling, click “Calculate Yields”, and then “Save” when the requester appears (to “pottery.yield” in the same directory).
    1. The yield results are transferred to the X-ray Spectrum Fit window.
  1. Close the Yield Calculation window.

 

X-ray Spectrum Fit

  1. The shape and position of the scatter peaks can vary and upset energy calibration fitting – let’s exclude them …
  2. Drag View markers to cover the range 1.4 to 17 keV (i.e. excluding the strong scatter peaks). Click on “Use View” in the X-ray Spectrum Fit window to make this range the fit range.
  3. Enable “Cal On” in the X-ray Spectrum Fit window. Fit this by clicking on “Fit: One”.
    1. This varies the energy cal parameters in the fit, but ignores the strong scatter peaks, which may upset the fit of energy calibration parameters due to Compton energy variation or if the beam energy is slightly in error.
  1. Now extend the View range out to 20 keV, and disable “Cal On” and “FWHM”.
    1. We now have included the scatter peaks, but with Cal and peak width parameters fixed in the fit.
  2. Fit this by clicking on “Fit: One”.
  3. While illustrative, this procedure is not needed anymore (beyond version 8.8y). Simply keep the View range as desired, enable “Cal On” in the fit and click “Fit: One” (or indeed “Fit: All” as appropriate). GeoPIXE will process the fit in two passes, as described above.

 

Final fitted Pottery shard SXRF spectrum

 

  1. Note the new results in the Fit Results window (use the “SpectrumàWindowsàFit Results” menu”). View concentrations or detection limits (ppm) by changing the drop-list to “Conc” or “MDL”.

    1. Adjust the width and position of the Compton peak using the parameters “Spread” à 1.2 and “Shift”, which effectively tune the electron momentum distribution. Fit again; that’s better?

                                                               i.      Better still is “Spread” à 1.5 and “Shift” à -0.005.

F.  Correction of Yields for Spatial Composition Variation

Sample composition varies generally across an image area. This means that the yield model assumed in projecting the images using the DA method will only be an approximation to any given pixel. This section describes a method to correct model yields on a pixel-by-pixel basis based on an end-member analysis of the image using a set of end-members that adequately describe the range of variation in composition (at least for the heavy elements that strongly affect self-absorption and secondary fluorescence).

 

See C.G. Ryan, "Quantitative Trace Element Imaging using PIXE and the Nuclear Microprobe", International Journal of Imaging Systems and Technology [Special issue on Quantitative Imaging] 11 (2000) 219-230 and C.G. Ryan, E. van Achterbergh, C.J. Yeats, Tin Tin Win and G. Cripps, “Quantitative PIXE trace element imaging of minerals using the new CSIRO-GEMOC Nuclear Microprobe”, Nucl. Instr. Meth. B189 (2002) 400-407.

 

A similar issue relates to fitting the spectra of regions with some unknown average composition. An end-member analysis can also be done, as part of the fit, to determine the proportion of end-members (in this region associated with the spectrum under fit) in order to correct yields for composition and iterate towards an improved fit to composition in the region. See the fitting example F3 below.

 

Note: The phase decomposition method here aims to span composition space sufficiently to produce better estimates of absorption, which dominates the modelling of X-ray yields, in order to correct images towards quantitative results. It may not provide a complete mineralogical breakdown of the phases present in a sample, which is quite a different goal.

 

Example F1: Correction of Images for Composition Variation

Correct images for matrix (composition) effects using an end-member decomposition approach. This should be done before trying any MPDA examples, as MPDA builds on and is a refinement of the approach here.

  • Background on composition effects in DA and the composition correction.
  • Selection of end-members.
  • Construction of the transformation matrix.
  • Correction for composition spatial variation.
  • Transformation onto images of end-member phase proportion.

 

Background on Matrix Correction of images

  1. The calculation of SXRF or PIXE yields (counts per ppm per unit of beam flux, etc.) is sensitive to the composition of the sample. A high average atomic number sample matrix will generally have higher self-absorption and produce lower yields. But it depends on the energy of the fluoresced X-rays versus absorption edges caused by component elements in the sample matrix.
  2. The calculation of yields tends to follow the rule that the yields are dominated by the effects of self-absorption and vary roughly inversely with self-absorption. Hence, we can use “the Bragg Rule” and combine inverse yields (1/Y) for the end-members in proportion to their mass fraction.
  3. DA imaging assumes a uniform sample/matrix composition. Hence, in some phases there may be significant errors in concentration of many elements. This is what we aim to correct here.
  4. After producing DA images assuming some uniform matrix composition, the images provide a first approximation to the composition variation, at least of the major elements that are visible via SXRF or PIXE. This allows us to estimate the proportion of the main end-member components in each pixel in order to improve the calculation of yields in each pixel. These improved yields can be used to converge on the true concentration images.

Background on MPDA

It uses the end-member components in each pixel to blend both the yields as well as intensities and background shape to better accumulate each event’s contribution to the images so that overlaps and background are subtracted more accurately.
  3. The ‘blend’ of terms uses the principal of averaging 1/yield terms in proportion to end-member weight fraction, to mirror the Bragg rule, as discussed in the Correction of Yields example (see Ryan (2000) paper in Help directory).

Background Notes

  1. The large size of the detector array, located just 10 mm from the beam spot on the sample, provides a large range of take-off angles at the sample surface, with the angle between sample normal and rays to masked detector element centre varying between 13.9° and 52.6°. This makes the outer detector elements much more sensitive to self-absorption for X-rays from a particle at depth in a sample than the inner detectors. Such a difference can be harnessed to provide an imaging contrast based on particle depth and also a quantitative measure of depth for discrete identified particles.

 

  1. The variation in model yields across the array is illustrated below, which shows the yield (counts/ppm/µC) of Pt Lα X-rays from a buried particle for individual detector elements plotted as a function of the radial distance of each detector from the centre of the array (beam axis). Each symbol type represents the yield from a 1 µm layer of Pt at various depths within an olivine sample matrix. Subtle absorption-related effects of particle shape versus a simple layer have been ignored in this discussion. The interesting apparent scatter in these points reflects the non-equivalent positions and orientations of detectors around the square array. Masking, distance and angle gradually reduces the efficiency of the more peripheral detector elements. Depth to the Pt layer has a marked effect on yields, with stronger attenuation seen in the more peripheral detectors.

Yield (counts/ppm/µC) of Pt Lα X-rays from a small Pt particle (modelled as a 1 µm buried Pt layer in olivine) versus radial position of individual Maia detector elements for various depths (µm) to the particle.

 

  1. If the two-layer sample model is correct, i.e. has the correct depth to the PGM, then both inner and outer detectors will “measure” the same effective concentration of Pt in the buried or “unknown” layer. Hence, since conc = area/yield, then (area/yield)inner = (area/yield)outer. Or, we can relate (areainner/areaouter) = (yieldinner/yieldouter). Therefore, we can relate the Ratio = (yieldinner/yieldouter) to the measured ratio (areainner/areaouter). They should be equal at the correct depth.

K.  Pileup effects in spectra and images

Example K1: Pileup effects in spectra - modelling pileup for spectra from regions

Understanding pileup effects in spectra and refining spectrum fitting

  • Pileup effects are quadratic, increasing like count-rates squared.
  • The pileup spectrum will reflect the count-rate products of the strongest lines in the spectrum.
  • A spectrum extracted from a region of a scan (or whole scan) may combine pixels with very different pileup.
  • How do we model pileup in spectra from such mixed regions?

 

Background Notes

The pulse train from the detector consists of quasi-Gaussian pulses arriving at random times. Pulses too close together “pile up”, meaning that the combined amplitude is digitized by the data acquisition system and appears at a higher apparent energy in the spectrum. The data acquisition system can flag and reject many of these pulses. However, it has a finite timing resolution, meaning it can only detect and reject the close pulses outside of some finite resolving time (t), typically around 60-200 ns. Within that time, we just see the “sum peak” at an energy corresponding to the sum of the energies of the contributing pulses, with a rate pN1N2t, where ‘N1’ and ‘N2’ are the count-rates of the lines and ‘p’ is a binomial probability factor. Hence, most systems with an active pileup rejection scheme still produce spectra with these “sum peaks”, which can masquerade as other elements of interest and cause problems for spectrum analysis or contaminate element images. It is these sum peaks that we seek to cater for here. The methods here are not suitable for a primitive system that includes no pileup rejection at all.

 

The spectrum of sum peaks is related to the product of the intensities of the strong lines in a spectrum. But with scanning of a sample through a beam, these strong lines can change for each pixel of the scan. Hence, the sum spectrum in each pixel is related to the product of the strong lines in that pixel, which may be different to the next pixel. Fortunately, our element images give us a handle on the strength of element lines in each pixel and so we can use the pixel-by-pixel product of images to estimate the strength of the pileup of the corresponding elements in each pixel. This approach can be used to estimate and correct pileup effects in images (see next example) or to build a better model of pileup for a mixed spectrum that was extracted from a general region of a scan, which may have different pileup in each pixel.

 

The methods outlined here are detailed in “C.G. Ryan, E. van Achterbergh and D.N. Jamieson, 2005, “Advances in Dynamic Analysis PIXE Imaging: Correction for Spatial Variation of Pileup Components”, Nucl. Instr. Meth. B231, 162-169” and in the Scenarios section “Fitting Spectra for Images with Complex Pile-up” in the GeoPIXE Users Guide.
